## What Python Is Responsible For

Python is the **intelligence engine**. It owns execution graphs, condition strategies, the caching layer, the compiler, the registry, the system setup/config layer, and the server-side logic for the Interaction API. TypeScript talks to Python via HTTP. Python must never assume the caller is a human — it is always another service.

| Responsibility | Description | Primary endpoint(s) |
|---|---|---|
| Concept execution | Run the execution graph for a concept against an entity | POST /execute, POST /execute/graph |
| Condition evaluation | Apply strategy + reference frame to a concept result | POST /evaluate/condition, POST /evaluate/full |
| Action triggering | Fire best-effort actions bound to conditions | POST /actions/{id}/trigger |
| Compilation | Transform semantic definitions into ExecutionGraphs (IR) | POST /compile, POST /compile/semantic |
| Registry governance | Write, version, promote, deprecate, diff definitions | POST /registry/definitions, /deprecate, /promote |
| Feature registry | Register, deduplicate, and track feature usage | POST /registry/features, GET .../usages |
| Caching layer | Cache Result (Rₜ) by (id, version, entity, timestamp) | Transparent to callers |
| Semantic hashing | Compute meaning_hash, semantic_hash, ir_hash | Internal to compiler |
| Async job queue | Accept, execute, and return long-running jobs | POST /execute/async, GET/DELETE /jobs/{id} |
| LLM refinement loop | Generate, refine, and validate concept definitions | POST /agents/*, POST /agents/semantic-refine |
| System setup (config) | Parse and apply memintel.config.md; register primitives, connectors, LLM, application context | `memintel apply` CLI + config API |
| Interaction API backend | Server-side logic for tasks, conditions, calibration, feedback | POST /tasks, GET/PATCH/DELETE /tasks/*, POST /conditions/*, POST /decisions/explain, POST /feedback/decision |

## Condition Strategies — All Six Are Required

Condition strategies are the core intelligence of the φ layer. Python is the **only** place these are implemented. TypeScript must never contain strategy logic.

### Strategy Ownership — Critical Rule

Strategies are NOT predefined by the admin, not hardcoded, and not assumed.
Every condition generated by the LLM during `POST /tasks` MUST include explicit strategy definition.

```python
# REQUIRED: Every condition object produced by the LLM must have both fields:
condition = {
    ...
    "strategy": {
        "type": "threshold",   # REQUIRED — must be one of the 6 registered types
        "params": {            # REQUIRED — fully specified, matching strategy schema
            "direction": "above",
            "value": 0.85
        }
    }
}
# If strategy.type or strategy.params is absent → compiler raises semantic_error
# There is no default strategy. The LLM must always resolve explicitly.
# Guardrails provide priors and bounds, but the LLM must output a complete strategy.
```

```python
from enum import Enum
from abc import ABC, abstractmethod

class StrategyType(str, Enum):
    THRESHOLD  = 'threshold'
    PERCENTILE = 'percentile'
    Z_SCORE    = 'z_score'    # MUST be 'z_score' — NOT 'zscore'. API, YAML, and guardrails all use z_score.
    CHANGE     = 'change'
    EQUALS     = 'equals'     # For categorical/string conditions
    COMPOSITE  = 'composite'  # For multi-factor AND/OR conditions

class ConditionStrategy(ABC):
    @abstractmethod
    def evaluate(
        self,
        result: Result,           # current concept result (Rₜ)
        history: list[Result],    # historical results for reference frame
        params: dict,
    ) -> DecisionValue: ...       # Returns DecisionValue, not raw bool — carries provenance
```

### THRESHOLD
Fires when the concept value crosses a fixed cutoff.

```python
# params: { direction: 'above' | 'below', value: float }
# Note: parameter key is 'value', NOT 'cutoff'. Guardrails and YAML both use 'value'.
# Use for: churn > 0.8, latency > 500ms, days_inactive > 14
# Logic: direction='above' → value > params['value']; direction='below' → value < params['value']
# Output type: decision<boolean>
```

### PERCENTILE
Fires when the value ranks above or below a percentile in historical results for the same entity.

```python
# params: { direction: 'top' | 'bottom', value: float (0-100) }
# Note: direction values are 'top'/'bottom', NOT 'above'/'below'.
# Use for: top 10% LTV, bottom 25th percentile retention
# Logic: rank result.value against history within window; compare rank to params['value']
# Output type: decision<boolean>
```

### Z_SCORE
Fires when deviation from the rolling mean exceeds a threshold.

```python
# params: { threshold: float, direction: 'above' | 'below' | 'any', window: duration }
# Note: parameter key is 'threshold' (not 'cutoff') for this strategy only.
# Use for: DAU anomaly, payment failure spike, volatility spike
# Logic: z = (value - mean(history)) / std(history); compare |z| to params['threshold']
# Output type: decision<boolean>
```

### CHANGE
Fires when the value has changed by more than X% relative to a previous value or window average.

```python
# params: { direction: 'increase' | 'decrease' | 'any', value: float, window: duration }
# Note: direction values are 'increase'/'decrease'/'any'. Parameter key is 'value'.
# Use for: stock price movement, engagement drop, metric spike
# Logic: pct_change = (current - previous) / previous; compare abs(pct_change) to params['value']
#        direction filters which direction of change fires the condition.
# Output type: decision<boolean>
```

### EQUALS
Fires when a categorical value matches one or more declared labels.

```python
# params: { value: string, labels: list[str] | None }
# Use for: classification outputs, status-based conditions, segment checks
# Logic: if labels is set, value must be in labels (subset of categorical type's label set).
#        Fire when result.value == params['value'].
# Input types: categorical, string ONLY. Raises type_error on float/int/boolean input.
# Output type: decision<categorical>
#
# CALIBRATION NOTE: equals has no numeric parameter to adjust.
# POST /conditions/calibrate MUST return status='no_recommendation' with
# reason='not_applicable_strategy' for any condition using this strategy.
# Do NOT attempt parameter adjustment. Log that feedback was stored but calibration
# is not applicable.
```

### COMPOSITE
Combines two or more decision<boolean> outputs using logical operators.

```python
# params: { operator: 'AND' | 'OR', operands: list[condition_id] }
# Use for: multi-factor decisions, (high_churn AND high_value) OR critical_event
# Logic: resolve each operand condition, apply operator across results.
# Input types: decision<boolean> ONLY. Composite cannot wrap equals conditions.
# Output type: decision<boolean>
#
# COMPILER ENFORCEMENT — these violations must be caught at compile time:
#   - operands must all produce decision<boolean>
#   - equals conditions produce decision<categorical> → type_error if used as operand
#   - composite cannot be nested inside another composite's operands → semantic_error
#   - operands list must contain at least 2 condition_ids → semantic_error if fewer
#   - all operand condition_ids must exist in the registry → reference_error if not
#   - violation message must identify the offending operand condition_id
#
# EXECUTION ORDER for composite:
#   1. Evaluate each operand condition independently (may be parallelized)
#   2. Apply operator (AND | OR) across the boolean results
#   3. Return a single decision<boolean>
#
# BIAS NOTE: Parameter bias rules do NOT apply to composite strategies directly.
# Bias applies only to the underlying operand conditions when those are generated.
```

## Type System Enforcement (Compiler)

The compiler must enforce all type rules from `memintel_type_system.md` v1.1 on every compile pass. Violations halt compilation with a `type_error`.

```python
# Supported base types (v1.1):
#   Scalar:    float, int, boolean, string, categorical
#   Container: time_series<float>, time_series<int>, list<float>, list<int>
#   Modifier:  T? (nullable — any type wrapped as Optional)
#   Decision:  decision<boolean>, decision<categorical>
#   Parameter: duration (compile-time only — NOT a node output type)

# Subtype rules:
#   int → float        (widening, implicit)
#   T → T?             (nullable widening, implicit)
#   time_series<int> → time_series<float>  (via int→float)
#   list<int>        → list<float>          (via int→float)
#   float → int        REQUIRES explicit to_int() — raises type_error otherwise
#   T? → T             REQUIRES null-handling operator — raises type_error otherwise

# Rule 1: Operator input validation
#   Each operator declares required input types. Mismatches → type_error.
#   Example: pct_change requires time_series<float>. Passing float → type_error.

# Rule 2: Output type propagation
#   Compiler infers output type of each node and passes to downstream nodes.
#   time_series<float> → pct_change → float → z_score_op → float  (valid)

# Rule 3: No implicit casting (except subtype widening above)
#   float ≠ time_series<float>. add(time_series, float) → type_error

# Rule 4: Condition input rule
#   threshold/percentile/z_score/change: accept float, int (via subtype)
#   equals: accepts categorical, string ONLY
#   composite: accepts decision<boolean> ONLY
#   No condition strategy accepts time_series, list, or duration as input.

# Rule 5: Condition output rule
#   boolean strategies → decision<boolean>
#   equals → decision<categorical>
#   decision<T> does NOT flow back into concept operators — only into actions
#   or unwrap_decision().

# Rule 6: duration is parameter-only
#   duration cannot be a node output type. Using duration as output → type_error.

# Rule 7: categorical label enforcement
#   categorical type must declare a closed label set at definition time.
#   Runtime values outside the declared set → type_error.
#   Two categorical types with different label sets are not compatible.

# Rule 8: nullable propagation
#   If any input to an operator is T?, the output is T? unless the operator
#   explicitly handles null (coalesce, drop_null, fill_null).
#   Primitives with no missing_data_policy produce T? output.

class TypeChecker:
    def check_node(self, node: GraphNode, input_types: dict[str, str]) -> str:
        """Returns the output type of the node, or raises type_error."""
        op = OPERATOR_REGISTRY[node.op]
        for inp in op.inputs:
            actual = input_types.get(inp.name)
            if not self.is_assignable(actual, inp.type):
                raise MemintelError(
                    'type_error',
                    f"Operator '{node.op}' expects '{inp.type}' for '{inp.name}', got '{actual}'"
                )
        return op.output.type

    def is_assignable(self, actual: str, expected: str) -> bool:
        """Returns True if actual type satisfies expected type (including subtype rules)."""
        if actual == expected:
            return True
        # int → float widening
        if expected == 'float' and actual == 'int':
            return True
        # T → T? nullable widening
        if expected.endswith('?') and actual == expected[:-1]:
            return True
        # time_series<int> → time_series<float>
        if expected == 'time_series<float>' and actual == 'time_series<int>':
            return True
        # list<int> → list<float>
        if expected == 'list<float>' and actual == 'list<int>':
            return True
        return False
```

## Deterministic Execution — Implementation Rules

```python
# ══════════════════════════════════════════════════════
# DETERMINISM CONTRACT — ALL EXECUTION PATHS MUST HONOUR
# ══════════════════════════════════════════════════════

# Rule 1: timestamp present → deterministic execution
# All data lookups MUST use the timestamp as the point-in-time reference.
# Fetch data 'as-of' the timestamp, not current state.
# result.deterministic MUST be True.

# Rule 2: same inputs → same output (always, forever)
# execute(id, version, entity, timestamp) MUST return identical results
# on every call, regardless of when the call is made, on any machine.
# This is not best-effort — it is a hard guarantee.

# Rule 3: timestamp absent → snapshot mode
# Use current data state. result.deterministic MUST be False.
# Do NOT cache snapshot results across requests.
# Snapshot results MUST NOT be cached at all beyond the current request context.

# Rule 4: cache key contract — EXACT FORM REQUIRED
# Cache key MUST be: (concept_id, version, entity, timestamp)
# A None timestamp IS A DIFFERENT KEY from any specific timestamp.
# Never combine or alias these — they represent different execution modes.

# Rule 5: ir_hash verification (execute_graph only)
# If ir_hash is supplied, verify it matches the stored graph before executing.
# Return HTTP 409 on mismatch. This is the audit trail mechanism.

# ══════════════════════════════════════════════════════
# NO LLM IN RUNTIME — ABSOLUTE RULE
# ══════════════════════════════════════════════════════
# The execution engine (execute, evaluate/condition, evaluate/full, execute/graph,
# execute/batch, execute/range, execute/async) MUST NEVER call the LLM.
# LLM is used ONLY in:
#   - POST /tasks (TaskAuthoringService)
#   - POST /agents/* endpoints
# Any LLM call in the execution path is a critical bug. There are no exceptions.

async def execute_graph(
    graph_id: str,
    entity: str,
    ir_hash: str | None = None,
    timestamp: str | None = None,
    **kwargs,
) -> Result:
    graph = await graph_store.get(graph_id)
    if graph is None:
        raise HTTPException(status_code=404, detail='Graph not found')
    if ir_hash and graph.ir_hash != ir_hash:
        raise HTTPException(status_code=409, detail='ir_hash mismatch — re-compile required')
    return await runtime.execute(graph, entity, timestamp, **kwargs)
```

## Async Job Queue — Pattern

```python
from enum import Enum

class JobStatus(str, Enum):
    QUEUED    = 'queued'
    RUNNING   = 'running'
    COMPLETED = 'completed'
    FAILED    = 'failed'
    CANCELLED = 'cancelled'

@app.post('/execute/async', status_code=202)
async def execute_async(req: ExecuteRequest) -> Job:
    job = await job_queue.enqueue(req)
    return Job(
        job_id=job.id,
        status=JobStatus.QUEUED,
        poll_interval_seconds=2,
    )

@app.get('/jobs/{job_id}')
async def get_job(job_id: str) -> JobResult:
    job = await job_store.get(job_id)
    if job is None:
        raise HTTPException(status_code=404)
    return JobResult(
        job_id=job.id,
        status=job.status,
        result=job.result if job.status == JobStatus.COMPLETED else None,
        error=job.error if job.status == JobStatus.FAILED else None,
    )

@app.delete('/jobs/{job_id}')
async def cancel_job(job_id: str) -> JobResult:
    job = await job_store.get(job_id)
    if job.status in (JobStatus.COMPLETED, JobStatus.FAILED):
        raise HTTPException(status_code=409, detail='Cannot cancel — job already completed or failed')
    await job_queue.cancel(job_id)
    return JobResult(job_id=job_id, status=JobStatus.CANCELLED)
```

## Action Execution — Best-Effort Contract

```python
# Actions are triggered automatically when ConditionEvaluator returns a DecisionValue.
# Action execution MUST NOT block the pipeline response.
# The pipeline returns HTTP 200 even if an individual action fails.
# Failure details appear in decision.actions_triggered[].status and .error.

# Execution types: webhook | notification | workflow | register
# Each action is bound to a specific condition version via trigger.condition_id.

# fire_on controls when the action fires:
#   'true'  → fires when decision.value is True (or matched label for equals)
#   'false' → fires when decision.value is False
#   'any'   → fires on every evaluation

async def trigger_bound_actions(
    decision: DecisionValue,
    condition: ConditionDefinition,
    dry_run: bool = False,
) -> list[ActionTriggered]:
    triggered = []
    for action in condition.bound_actions:
        if not _should_fire(action.trigger.fire_on, decision.value):
            triggered.append(ActionTriggered(action_id=action.id, status='skipped'))
            continue
        if dry_run:
            triggered.append(ActionTriggered(action_id=action.id, status='would_trigger'))
            continue
        try:
            result = await action_executor.trigger(action, decision)
            triggered.append(ActionTriggered(action_id=action.id, status='triggered',
                                             payload_sent=result.payload_sent))
        except Exception as e:
            triggered.append(ActionTriggered(action_id=action.id, status='failed',
                                             error=ErrorResponse(type='execution_error', message=str(e))))
    return triggered
```

## System Setup — `memintel apply` Implementation

The `memintel apply` command is the admin entry point. It reads `memintel.config.md`, validates it, and registers all resources with the backend. It also loads `memintel.guardrails.md` and stores the application_context block for injection into LLM prompts during task authoring.

```python
class ConfigApplyService:
    async def apply(self, config_path: str) -> ApplyResult:
        # Step 1: Parse Markdown → extract YAML blocks
        raw_yaml = self.extract_yaml(config_path)

        # Step 2: Validate config schema
        config = ConfigSchema.model_validate(raw_yaml)

        # Step 3: Register primitives
        for primitive in config.primitives:
            await self.primitive_registry.register(primitive)

        # Step 4: Register connectors (credentials stored securely, never logged)
        for name, connector in config.connectors.items():
            connector_resolved = self.resolve_env_vars(connector)
            await self.connector_registry.register(name, connector_resolved)

        # Step 5: Store environment config
        await self.env_store.set(config.environment)

        # Step 6: Configure LLM endpoint
```

```python
class CreateTaskRequest(BaseModel):
    intent: str
    entity_scope: str
    delivery: DeliveryConfig
    constraints: ConstraintsConfig | None = None
    dry_run: bool = False

class TaskAuthoringService:
    async def create_task(self, req: CreateTaskRequest) -> Task:
        # 1. Load full guardrails context for LLM injection
        #    This is the most critical step — the LLM must NOT freely interpret intent.
        #    It must resolve strategy and parameters within the guardrails constraints.
        guardrails = await self.guardrails_store.get()
        app_context = await self.app_context_store.get()

        # 2. Build LLM prompt context — strict injection order:
        #    [1] Type system summary (hard rules)
        #    [2] Guardrails (strategy registry, type_compatibility, priors, bias rules)
        #    [3] Application context (domain description + instructions)
        #    [4] Parameter bias rules (deterministic instruction → severity shift mappings)
        #    [5] Primitive registry (available data signals)
        llm_context = self.build_llm_context(guardrails, app_context)

        # 3. Call LLM with full context (using configured LLM from memintel.config.md)
        #    LLM must follow guardrails priority order:
        #    user_explicit > primitive_hint > mapping_rule > application_context >
        #    global_preferred > global_default
        concept_def = await self.llm.generate_concept(
            intent=req.intent,
            entity_scope=req.entity_scope,
            context=llm_context,
        )
        condition_def = await self.llm.generate_condition(
            concept=concept_def,
            constraints=req.constraints,
            context=llm_context,
        )
        # Action binding resolution order:
        #   1. User-specified (from intent or delivery config)
        #   2. Guardrails default_action
        #   3. application_context.action_preferences (by resolved severity)
        #   4. System default
        #   5. Fail if unresolved — return HTTP 422 with action_binding_failed
        action_def = await self.llm.resolve_action(
            delivery=req.delivery,
            guardrails=guardrails,
            app_context=app_context,
            condition=condition_def,
        )
        if action_def is None:
            raise HTTPException(status_code=422, detail={
                'error': {'type': 'action_binding_failed', 'message': 'No action could be resolved'}
            })

        # 4. Validate all three definitions through compiler
        for defn in [concept_def, condition_def, action_def]:
            result = await self.compiler.validate(defn)
            if not result.valid:
                # Enter refinement loop (up to N retries)
                defn = await self.llm.refine(defn, result.errors, context=llm_context)
                # Re-validate after refinement...

        # LLM FAILURE HANDLING
        # If the refinement loop fails to produce a valid definition after MAX_RETRIES:
        #   → raise HTTP 422 with error.type = 'semantic_error'
        #   → include the last compiler validation error in the response
        #   → do NOT persist any partial definition — no task_id is ever assigned
        #   → system remains clean as if the request was never made
        # MAX_RETRIES is implementation-defined; recommended value: 3
        # On each retry: pass only the failing definition + its errors back to the LLM.
        # Instruct the LLM to fix only the invalid parts, not regenerate from scratch.

        if req.dry_run:
            # Return preview — do NOT register, compile, or persist anything
            return Task(
                task_id=None,
                status='preview',
                concept_id=concept_def.id,
                concept_version=concept_def.version,
                condition_id=condition_def.id,
                condition_version=condition_def.version,
                action_id=action_def.id,
                action_version=action_def.version,
                intent=req.intent,
                entity_scope=req.entity_scope,
                delivery=req.delivery,
            )

        # 5. Register all three definitions
        ns = req.constraints.namespace if req.constraints else 'org'
        await self.registry.register(concept_def, namespace=ns)
        await self.registry.register(condition_def, namespace=ns)
        await self.registry.register(action_def, namespace=ns)

        # 6. Compile concept to execution graph
        await self.compiler.compile(concept_def.id, concept_def.version)

        # 7. Persist task (version-pinned — never auto-updates)
        task = await self.task_store.create(Task(
            task_id=generate_id(),
            status='active',
            intent=req.intent,
            entity_scope=req.entity_scope,
            delivery=req.delivery,
            concept_id=concept_def.id,
            concept_version=concept_def.version,
            condition_id=condition_def.id,
            condition_version=condition_def.version,
            action_id=action_def.id,
            action_version=action_def.version,
            created_at=utc_now(),
        ))

        return task
```

### POST /conditions/calibrate

```python
# Calibration is the bridge between stored feedback and parameter adjustment.
# It does NOT modify any live condition — it returns a recommendation + token.
# The caller decides whether to apply (via POST /conditions/apply-calibration).

class CalibrateRequest(BaseModel):
    condition_id: str
    condition_version: str
    target: TargetConfig | None = None      # { alerts_per_day: float }
    feedback_direction: str | None = None   # 'tighten' | 'relax' — explicit override

class CalibrationService:
    async def calibrate(self, req: CalibrateRequest) -> CalibrationResult:
        condition = await self.registry.get(req.condition_id, req.condition_version)

        # EQUALS strategy — no numeric parameter to adjust
        if condition.strategy.type == StrategyType.EQUALS:
            return CalibrationResult(
                status='no_recommendation',
                no_recommendation_reason='not_applicable_strategy',
                current_params=condition.strategy.params,
            )

        # Resolve feedback direction
        if req.feedback_direction:
            direction = req.feedback_direction   # explicit override
        else:
            # Aggregate stored feedback for this condition version
            feedback_records = await self.feedback_store.get_by_condition(
                req.condition_id, req.condition_version
            )
            direction = self.derive_direction(feedback_records)
            # direction = 'tighten' if majority false_positive
            # direction = 'relax'   if majority false_negative
            # direction = None      if insufficient data

        if direction is None:
            return CalibrationResult(
                status='no_recommendation',
                no_recommendation_reason='insufficient_data',
                current_params=condition.strategy.params,
            )

        # Compute recommended params — strategy-aware, not threshold-specific
        recommended = self.adjust_params(
            strategy=condition.strategy.type,
            current_params=condition.strategy.params,
            direction=direction,
            target=req.target,
            guardrails=await self.guardrails_store.get(),
        )

        if recommended is None:
            return CalibrationResult(
                status='no_recommendation',
                no_recommendation_reason='bounds_exceeded',
                current_params=condition.strategy.params,
            )

        # Estimate impact
        impact = await self.estimate_impact(condition, recommended)

        # Generate single-use token (expires 24h)
        token = await self.token_store.create(CalibrationToken(
            condition_id=req.condition_id,
            condition_version=req.condition_version,
            recommended_params=recommended,
            expires_at=utc_now_plus(hours=24),
        ))

        return CalibrationResult(
            status='recommendation_available',
            recommended_params=recommended,
            calibration_token=token,
            current_params=condition.strategy.params,
            impact=impact,
        )

    def adjust_params(
        self,
        strategy: StrategyType,
        current_params: dict,
        direction: str,
        target: TargetConfig | None,
        guardrails: Guardrails,
    ) -> dict | None:
        """
        Returns adjusted params dict, or None if adjustment would violate bounds.
        Strategy-aware — uses bias_semantics from guardrails:
          threshold:  relax → decrease 'value';  tighten → increase 'value'
          percentile: relax → increase 'value';  tighten → decrease 'value'
          change:     relax → decrease 'value';  tighten → increase 'value'
          z_score:    relax → decrease 'threshold'; tighten → increase 'threshold'
        """
        bounds = guardrails.constraints.threshold_bounds.get(strategy.value, {})
        on_exceeded = guardrails.constraints.on_bounds_exceeded  # 'clamp' | 'reject'

        adjusted = dict(current_params)
        param_key = 'threshold' if strategy == StrategyType.Z_SCORE else 'value'
        current_val = current_params.get(param_key, 0.0)

        # Apply adjustment (simplified — real implementation uses target alerts_per_day)
        delta = self.compute_delta(strategy, direction, current_val, target)
        new_val = current_val + delta

        # Bounds check
        min_val = bounds.get('min')
        max_val = bounds.get('max')
        if min_val is not None and new_val < min_val:
            if on_exceeded == 'reject':
                return None
            new_val = min_val   # clamp
        if max_val is not None and new_val > max_val:
            if on_exceeded == 'reject':
                return None
            new_val = max_val   # clamp

        adjusted[param_key] = new_val
        return adjusted
```

```python
# Creates a new immutable condition version from a calibration token.
# MUST NOT auto-rebind tasks — tasks_pending_rebind is informational only.
# Caller rebinds explicitly via PATCH /tasks/{id}.

class ApplyCalibrationRequest(BaseModel):
    calibration_token: str        # required — single-use, resolves all params server-side
    new_version: str | None = None

    # IMPORTANT: Do NOT add condition_id, condition_version, threshold, or
    # rebind_tasks to this model. The token is the only path. There is no
    # fallback of passing explicit params. The token resolves everything.

class ApplyCalibrationService:
    async def apply(self, req: ApplyCalibrationRequest) -> ApplyCalibrationResult:
        # Resolve calibration token
        token = await self.token_store.resolve(req.calibration_token)
        if token is None or token.is_expired():
            raise HTTPException(status_code=400, detail={
                'error': {'type': 'parameter_error', 'message': 'Invalid or expired calibration token'}
            })

        condition_id = token.condition_id
        source_version = token.condition_version
        recommended_params = token.recommended_params   # generic dict, not threshold-specific

        # Create a NEW condition version — never mutate the existing one
        source = await self.registry.get(condition_id, source_version)
        new_version = req.new_version or self.auto_increment_version(source_version)

        if await self.registry.exists(condition_id, new_version):
            raise HTTPException(status_code=409, detail={
                'error': {'type': 'conflict', 'message': 'Version already exists'}
            })

        # Apply recommended_params to a copy of the condition strategy
        new_condition = source.copy_with_params(recommended_params, new_version)
        await self.registry.register(new_condition)

        # Invalidate the token — single-use
        await self.token_store.invalidate(req.calibration_token)

        # Find tasks still bound to the old version — for informational purposes ONLY
        # DO NOT rebind automatically. Caller must rebind via PATCH /tasks/{id}.
        tasks_pending = await self.task_store.find_by_condition_version(
            condition_id, source_version
        )

        return ApplyCalibrationResult(
            condition_id=condition_id,
            previous_version=source_version,
            new_version=new_version,
            params_applied=recommended_params,
            tasks_pending_rebind=[
                {'task_id': t.task_id, 'intent': t.intent}
                for t in tasks_pending
            ],
        )

    # INVARIANT: tasks_pending_rebind is informational. Python MUST NOT
    # auto-rebind tasks here under any circumstances.
    # The old 'rebind_tasks' parameter and auto-rebind branch are REMOVED.
```

### POST /feedback/decision

```python
# Feedback is stored as structured records. It does NOT modify any live condition.
# Feedback is an input to POST /conditions/calibrate — the recommended workflow:
# accumulate feedback → calibrate → apply-calibration → rebind tasks

class FeedbackValue(str, Enum):
    FALSE_POSITIVE = 'false_positive'   # condition fired but should not have → tighten
    FALSE_NEGATIVE = 'false_negative'   # condition did not fire but should have → relax
    CORRECT        = 'correct'          # expected behavior → no calibration adjustment

class FeedbackRecord(BaseModel):
    feedback_id: str
    condition_id: str
    condition_version: str
    entity: str
    timestamp: str         # timestamp of the original decision
    feedback: FeedbackValue
    note: str | None = None
    recorded_at: str       # ISO 8601 UTC — when this feedback was submitted

# Validation:
# - Must verify a decision exists for condition_id + entity + timestamp.
# - Return HTTP 404 if no decision found at that timestamp.
# - feedback must be one of: false_positive, false_negative, correct.
#   'useful', 'not_useful' are NOT valid values.
#
# FEEDBACK DEDUPLICATION — uniqueness key:
#   (condition_id, condition_version, entity, timestamp)
# - If a feedback record already exists for this exact key:
#   → return HTTP 409, error.type = 'conflict'
#   → message: 'Feedback already submitted for this decision'
# - This uniqueness is enforced at DB level (unique constraint) AND here
# - Rationale: duplicate submissions bias calibration direction counts.
#   3 real false_positives + 6 duplicates of the same one will produce
#   an overly aggressive calibration recommendation.

# CalibrationService.derive_direction() uses feedback records:
def derive_direction(self, records: list[FeedbackRecord]) -> str | None:
    if len(records) < MIN_FEEDBACK_THRESHOLD:
        return None
    fp_count = sum(1 for r in records if r.feedback == FeedbackValue.FALSE_POSITIVE)
    fn_count = sum(1 for r in records if r.feedback == FeedbackValue.FALSE_NEGATIVE)
    if fp_count > fn_count:
        return 'tighten'
    if fn_count > fp_count:
        return 'relax'
    return None   # tie → insufficient signal
```

```python
    strategy_type=condition.strategy.type,
    params_applied=condition.strategy.params,  # generic dict
    actions_triggered_count=len(decision.actions_triggered),
)

# CALIBRATION LOGS — every calibrate and apply must log:
log.info("calibration_recommended",
    condition_id=condition_id,
    condition_version=condition_version,
    strategy_type=strategy_type,
    old_params=current_params,
    recommended_params=recommended_params,
    delta_alerts=impact.delta_alerts if impact else None,
    feedback_direction=direction,
)

log.info("calibration_applied",
    condition_id=condition_id,
    previous_version=previous_version,
    new_version=new_version,
    params_applied=params_applied,
    tasks_pending_rebind=len(tasks_pending_rebind),
)

# ERROR LOGS — every raised MemintelError must log:
log.error("memintel_error",
    error_type=e.type,          # machine-readable
    location=e.location,        # where in definition the error occurred
    entity=entity,              # if available
    concept_id=concept_id,      # if available
    condition_id=condition_id,  # if available
)

# LOGGING RULES:
# ✅ Log concept_id, version, entity, timestamp on every execution
# ✅ Log decision_value, strategy_type, params_applied on every decision
# ✅ Log old_params → new_params on every calibration
# ✅ Log error.type and location on every error
# ❌ NEVER log resolved credential values (${ENV_VAR} values)
# ❌ NEVER log raw primitive data or entity attribute values
# ❌ NEVER log feedback note fields (may contain PII)
# ❌ Do not use print() — use structured logging only
```

## What Python Must NOT Do

```python
# ❌ DO NOT return different results for the same (id, version, entity, timestamp)
#    Determinism is a hard requirement, not best-effort.

# ❌ DO NOT execute actions synchronously in the critical path
#    Actions are best-effort and must not block the pipeline HTTP response.

# ❌ DO NOT implement camelCase field names in Pydantic models
#    All internal and API-facing models use snake_case.

# ❌ DO NOT resolve 'latest' version implicitly
#    Every execution call requires an explicit version string.
#    Return HTTP 400 if version is missing or 'latest'.

# ❌ DO NOT skip Pydantic validation for incoming requests
#    All requests enter the system through validated Pydantic models.

# ❌ DO NOT implement LLM logic directly in route handlers
#    Agent endpoints and task authoring delegate to isolated service classes.

# ❌ DO NOT expose Internal Platform endpoints without elevated key check
#    /compile, /execute/graph, /registry (write), /agents/semantic-refine, etc.
#    Gate them with a permission check on the API key before routing.

# ❌ DO NOT mutate registered definitions
#    Definitions are immutable. Updates require a new version.
#    Return HTTP 409 if id+version already exists.

# ❌ DO NOT auto-rebind tasks when a new condition version is created
#    Tasks remain pinned to their current condition version until explicitly rebound
#    via PATCH /tasks/{id}. apply-calibration returns tasks_pending_rebind as
#    informational only — it does NOT rebind. There is no rebind_tasks parameter.

# ❌ DO NOT allow partial config application in memintel apply
#    Either all steps succeed or all are rolled back. No partial state.

# ❌ DO NOT log resolved credential values from config
#    ${ENV_VAR} references must be resolved but never written to logs or traces.

# ❌ DO NOT retry failed actions automatically
#    Actions are at-most-once per evaluation call (best-effort contract).
#    A failed action is captured in actions_triggered[].status='failed'.
#    The pipeline returns HTTP 200 regardless. No automatic retry.
#    Downstream systems are responsible for their own idempotency.
#    Recommended deduplication key for downstream:
#      (condition_id, condition_version, entity, timestamp)

# ❌ DO NOT call the LLM in any execution path — this is an ABSOLUTE RULE
#    The execution engine (execute, evaluate/condition, evaluate/full, execute/graph,
#    execute/batch, execute/range, execute/async) MUST NEVER invoke the LLM.
#    Any LLM call in an execution endpoint is a critical bug with no exceptions.
#    LLM is restricted to: TaskAuthoringService (POST /tasks) and POST /agents/* only.

# ❌ DO NOT call the LLM without injecting the full guardrails context
#    POST /tasks must load memintel.guardrails.md and application_context before
#    any LLM call. The LLM must never freely interpret intent — it resolves
#    strategy and parameters within deterministic guardrail constraints.

# ❌ DO NOT use 'useful' or 'not_useful' as feedback values
#    The only valid FeedbackValue values are: false_positive, false_negative, correct.

# ❌ DO NOT use strategy-specific field names in generic parameter handling
#    Use recommended_params: dict and params_applied: dict everywhere.
#    The old 'threshold_applied' and 'copy_with_threshold()' patterns are removed.
#    Use 'copy_with_params(params: dict, version: str)' instead.

# ❌ DO NOT implement CHANGE strategy as a crossing-only detector by default
#    CHANGE fires when the value has changed by more than X% relative to a previous
#    value or window average. It is not limited to crossing events only.
#    If crossing-only semantics are needed, that is a separate strategy variant.
```

## Error Handling Pattern

```python
from memintel.exceptions import MemintelError

try:
    result = await client.execute(id='org.churn_risk', version='1.0', entity='user_abc')
except MemintelError as e:
    match e.type:
        case 'not_found':
            print(f'Check id/version: {e.message}')
        case 'semantic_error':
            print(f'Semantic issue at {e.location}: {e.message}')
            print(f'Fix: {e.suggestion}')
        case 'execution_error':
            print(f'Data issue at {e.location}: {e.message}')
        case 'execution_timeout':
            # Switch to async
            job = await client.execute_async(id='org.churn_risk', version='1.0', entity='user_abc')
        case 'rate_limit_exceeded':
            import time; time.sleep(e.retry_after_seconds)
        case 'bounds_exceeded':
            print(f'Calibration hit guardrail bound: {e.message}')
        case 'action_binding_failed':
            print(f'No action resolved: {e.suggestion}')
        case _:
            raise

# Valid ErrorType values (align with memintel_api.yaml ErrorResponse enum):
#   syntax_error, type_error, semantic_error, reference_error, parameter_error,
#   graph_error, execution_error, execution_timeout, auth_error, not_found,
#   conflict, rate_limit_exceeded, bounds_exceeded, action_binding_failed
#
# 'no_recommendation' is NOT an error type — it is CalibrationResult.status.
# Do NOT raise MemintelError with type='no_recommendation'.
```

```bash
pip install memintel-sdk

pip install "memintel-sdk[async]"
```

Client initialisation:

```python
import memintel

# Explicit key
client = memintel.Client(api_key='sk-...')

# Auto-read from environment (recommended)
client = memintel.Client()   # reads MEMINTEL_API_KEY

# Async client
async_client = memintel.AsyncClient()
```

Package must export:
- `memintel.Client` — synchronous client
- `memintel.AsyncClient` — async client
- `memintel.exceptions.MemintelError` and all typed subclasses
- `memintel.models.*` — all Pydantic models for public schemas
- `memintel.cli` — `memintel apply` CLI entrypoint
